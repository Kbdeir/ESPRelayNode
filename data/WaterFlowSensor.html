Sensor Settings

Higher values reduce the reported flow rate; lower values increase it.

Summary

Connection

  • Signal wire connects to Input 6 / GPIO27.
  • The firmware enables INPUT_PULLUP on this pin.
  • Sensor output should pulse low; pulses are counted on the falling edge.
  • Power and ground must match the sensor module requirements.

Functionality

  • Each pulse interrupt increments an internal counter.
  • Every 5 seconds, the device calculates flow in litres per minute.
  • It also accumulates total water volume in litres since boot.
  • Readings are published to MQTT as retained messages.

MQTT Output

  • Sensor MQTT Topic receives the current flow rate.
  • The value is formatted with one decimal place, e.g. 12.4.
  • Total volume is published to the fixed topic TotalLiters.

Restrictions

  • Available only when the firmware is built with WaterFlowSensor.
  • Input 6 cannot be used as a normal input mode while this feature is enabled.
  • The accumulated total is saved to flash every 5 minutes and survives reboots.
Calibration converts pulse frequency into litres per minute. The default factor is 4.5, which is common for YF-style hall-effect flow sensors, but the best value depends on your exact sensor and plumbing.
