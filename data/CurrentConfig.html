HC-SR04 Sonar / Ultrasonic Distance Sensor ▼

🔌 Wiring & Pin Assignment

The HC-SR04 ultrasonic sensor uses two GPIO pins shared with the input header:

HC-SR04 PinGPIOHeader PinNote
VCC—5 V railSensor requires 5 V supply
GND—GND
TRIGGPIO17Input 33.3 V output — direct connection safe
ECHOGPIO32Input 4⚠ 5 V output — use voltage divider!

ECHO voltage divider (required)

The HC-SR04 ECHO pin outputs 5 V, but GPIO32 is a 3.3 V logic pin. Connect a 1 kΩ resistor in series then a 2 kΩ resistor to GND between the ECHO pin and GPIO32. This divides 5 V down to 3.33 V, which is within the safe input range.

Alternatively, use an HC-SR04+ (3.3 V variant) which can be powered at 3.3 V and outputs 3.3 V on ECHO — no divider needed.

Enabling sonar in firmware

Sonar is a compile-time feature. Uncomment #define SR04 in src/defines.h then rebuild and flash. The alternative serial-mode variant is #define SR04_SERIAL.

⚠
Input pin conflict — Sonar TRIG uses GPIO17 (Input 3) and ECHO uses GPIO32 (Input 4). When sonar is active these two inputs cannot be used for relay control, toggle switches, or any other input function. If you have devices wired to Input 3 or Input 4, disconnect them before enabling sonar.
